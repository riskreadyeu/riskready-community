# 🛠️ Component Scaffolding Guide

## Overview

This guide explains how to use the component scaffolding CLI to quickly generate standardized pages that follow the RiskReady design system.

---

## 🚀 Quick Start

### Generate a List Page
```bash
npm run scaffold:list

# Interactive prompts:
# - Template type: list
# - Entity name: Nonconformity
# - Module path: audits
# 
# Creates: apps/web/src/pages/audits/NonconformityRegisterPage.tsx
```

### Generate a Detail Page
```bash
npm run scaffold:detail

# Interactive prompts:
# - Template type: detail
# - Entity name: Nonconformity
# - Module path: audits
# 
# Creates: apps/web/src/pages/audits/NonconformityDetailPage.tsx
```

### Generate a Dashboard Page
```bash
npm run scaffold:dashboard

# Interactive prompts:
# - Template type: dashboard
# - Entity name: Audits
# - Module path: audits
# 
# Creates: apps/web/src/pages/audits/AuditsDashboardPage.tsx
```

## 🎨 Customization

### Modify Templates

Templates are located in:
```
scripts/templates/
├── list-page.template.js
├── detail-page.template.js
└── dashboard-page.template.js
```

Edit these files to customize the generated output for your organization's needs.

### Add New Templates

1. Create new template file: `scripts/templates/my-template.template.js`
2. Add template to `scripts/scaffold.js`:
   ```javascript
   const templates = {
     list: require('./templates/list-page.template'),
     detail: require('./templates/detail-page.template'),
     dashboard: require('./templates/dashboard-page.template'),
     mytemplate: require('./templates/my-template.template'),  // Add here
   };
   ```
3. Add npm script to `package.json`:
   ```json
   "scaffold:mytemplate": "node ../../scripts/scaffold.js mytemplate"
   ```

## 🔍 Troubleshooting

### Template Not Found
```bash
Error: Cannot find module './templates/list-page.template'
```
**Solution**: Ensure template files exist in `scripts/templates/` directory

### Directory Creation Failed
```bash
Error: ENOENT: no such file or directory
```
**Solution**: CLI will prompt to create directory. Answer 'y' to proceed.

### File Already Exists
```bash
File NonconformityRegisterPage.tsx already exists. Overwrite? (y/n):
```
**Solution**: 
- Type 'y' to overwrite
- Type 'n' to cancel and manually rename/move existing file

## 💡 Tips & Best Practices

1. **Always run from workspace root**
   ```bash
   cd /path/to/riskready-community
   npm run scaffold:list
   ```

2. **Use PascalCase for entity names**
   - ✅ `Nonconformity`
   - ✅ `Policy`
   - ❌ `nonconformity`
   - ❌ `policy`

3. **Use lowercase for module paths**
   - ✅ `audits`
   - ✅ `policies`
   - ❌ `Audits`

4. **Follow naming conventions**
   - List: `{Entity}RegisterPage.tsx`
   - Detail: `{Entity}DetailPage.tsx`
   - Dashboard: `{Module}DashboardPage.tsx`

5. **Complete all TODOs before testing**
   - Search for `// TODO:` comments
   - Replace placeholder data
   - Implement actual API calls
